## Phase 5：多用户 SaaS（条件触发）⏸️ 等待

> **触发条件**：有第 1 个真实外部用户准备使用时开始建设

### 背景与原则
- 当前为个人使用，完整用户系统需 3~4 周，在无外部用户前不合算（YAGNI 原则）
- 一旦触发，采用**云端代理架构**，彻底解决 API Key 安全问题

### 目标架构

```
[用户桌面App] ──登录验证──▶ [云端服务（你的服务器）]
                                    │
             [代理 Gaode API] ◀──── │ ────▶ [用户数据库]
                                    │       [额度系统]
[管理员后台（Web）] ──────────────▶ │       [操作记录]
```

### 功能清单

**用户端：**
- [ ] 邮箱注册（含邮件验证）+ 昵称设置
- [ ] 邮箱登录，记住上次邮箱（本地存储）
- [ ] 15天免密登录（JWT refresh token）
- [ ] 修改密码/昵称
- [ ] 查看 API 额度余量、明细、申请记录
- [ ] 额度耗尽时提示，引导申请

**管理员端（Web 界面）：**
- [ ] 管理员账号密码登录
- [ ] 查看/处理用户额度申请（通过/拒绝）
- [ ] 查看 API 总池余量（手动录入，暂无高德平台接口）
- [ ] 查看所有用户的分发记录和使用明细

**技术选型：**
- 云端服务：Python FastAPI（复用现有经验）
- 数据库：PostgreSQL（或 SQLite，取决于用户规模）
- 认证：JWT（access token 短效 + refresh token 15天）
- 部署：轻量 VPS（20元/月）
- 高德 Key：存于服务器环境变量，永不下发给客户端

## 已知技术风险

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| 高德 API 额度耗尽 | 无法获取 POI | 聚类策略已大幅降低调用次数；无 POI 时降级到城市级别 |
| 视频时区转换不准确 | 视频归入错误行程 | 优先用 GPS 定位时区 |
| PyInstaller 打包依赖 | 发布阻塞 | 预留专项测试时间 |
| moviepy 性能瓶颈 | 视频生成极慢 | 后台任务 + 进度条 + 可取消 |
| 多用户后 Key 被滥用 | API 费用异常 | Phase 5 采用代理架构彻底解决 |
