## Phase 2：后端核心模块（第 2-5 周）

### Week 2：元数据提取 + 数据库

**目标**：能扫描一个文件夹，提取所有照片/视频的时间和 GPS 信息存入数据库

- [ ] 实现 SQLite 数据库初始化（`app/core/database.py`）
- [ ] 创建 SQLAlchemy 数据模型（`app/models/`）
- [ ] 实现照片元数据提取（PIL + piexif）
- [ ] 实现视频元数据提取（mutagen + ffmpeg-python）
- [ ] 实现 UTC 时区转换（视频时间修正）
- [ ] 处理无 GPS / 无时间文件的逻辑
- [ ] 实现 `/api/scan` 接口（异步扫描 + 进度推送）
- [ ] 编写单元测试（pytest）

**里程碑**：扫描本地照片文件夹，结果正确存入 SQLite

### Week 3：逆地理编码模块

**目标**：为每张有 GPS 的照片解析出省/市/区/景点

- [ ] 集成 `reverse_geocoder`（国际离线）
- [ ] 集成 `fast-geocn`（国内离线，中文地名）
- [ ] 实现智能切换：中国坐标用 fast-geocn，其他用 reverse_geocoder
- [ ] 实现地理编码缓存（SQLite `geocode_cache` 表）
- [ ] 集成高德地图 API（在线增强，获取 POI 景点名）
- [ ] 实现 API 限流与重试机制
- [ ] 实现 `/api/geocode/batch` 接口（批量处理 + 进度）

**里程碑**：给定一批坐标，正确返回中文地址，国内/国际均测试通过

### Week 4：智能归档算法

**目标**：实现行程切分算法，生成文件夹命名方案

- [ ] 实现行程切分核心算法（`app/services/archive_service.py`）
  - 大行程边界（30 天阈值，可配置）
  - 小行程切分（地点变化 + 2h 阈值）
  - 同地点合并逻辑
  - GPS 漂移过滤
  - 地点回归处理
- [ ] 实现文件夹命名生成（`YYYY-MM_行程名 > 序号_地点_MMDD-MMDD`）
- [ ] 实现 `/api/archive/preview` 接口
- [ ] 实现 `/api/trips` CRUD 接口（支持用户手动调整）

**里程碑**：给定云南旅行素材，归档方案与预期（昆明x2、大理、丽江、香格里拉）完全一致

### Week 5：文件操作 + 备注写入

**目标**：安全地执行文件复制归档，并写入备注

- [ ] 实现文件复制操作（保留原文件）
- [ ] 实现操作日志记录
- [ ] 实现 `/api/archive/execute` 接口
- [ ] 实现 `/api/archive/cleanup` 接口（删除原文件）
- [ ] 实现照片备注写入（piexif UserComment，UTF-8 编码）
- [ ] 实现视频备注写入（mutagen ©cmt）
- [ ] 验证写入后手机读取效果（iOS/Android 均测试）
- [ ] 实现 `/api/media/thumbnail` 缩略图接口

**里程碑**：完整执行一次归档流程（扫描→解析→预览→执行→备注写入），结果正确

## 已知技术风险

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| 视频时区转换不准确 | 视频归入错误行程 | 优先用 GPS 定位时区，次选用户手动设置 |
| fast-geocn 中文地名不全 | 部分景点名称缺失 | 降级处理，显示"省/市"级别 |
| PyInstaller 打包依赖问题 | 发布阻塞 | 预留专项测试时间，考虑 Nuitka 备选 |
| moviepy 性能瓶颈 | 视频生成极慢 | 加进度条 + 后台任务，用户可取消 |
| Selenium/Playwright 地图截图不稳定 | 视频质量差 | 预研阶段验证，必要时改用 folium 静态渲染 |
